Loan Approval Prediction System

Explore Source →
Python Pandas scikit-learn FastAPI Pydantic
Status Active Development
Role ML Engineer
Duration 4 Weeks
Last Updated July 2026
Repository Explore Source →

01. Overview

A classification model and API designed to predict loan approval outcomes using applicant financial profiles.

02. Problem

Manual credit evaluation is slow and subjective. Automating classifications requires preprocessing numeric/categorical features and exposing predictions as a fast, type-safe API.

03. Approach

Created an end-to-end preprocessing and model pipeline with scikit-learn, validating inputs using Pydantic, and serving inference requests via a FastAPI web server.

04. System Design

Client Request (JSON) │ ▼ FastAPI Gateway │ ▼ Pydantic Validation (Type Verification) │ ▼ scikit-learn Pipeline (Scaling/Encoding) │ ▼ Tabular Classifier Model │ ▼ JSON Classification Response

05. Engineering Decisions

Selected scikit-learn for training to leverage its native Pipeline interface. Chose Pydantic schema validation inside FastAPI to reject invalid client inputs at the API entry point.

06. Challenges

Data leakage between folds occurred during separate categorical and numerical preprocessing. Solved by encapsulation inside unified ColumnTransformer and Pipeline flows.

07. Lessons Learned

Biggest Lesson (Model vs. System): Model accuracy wasn't the hardest problem. Building a reliable preprocessing pipeline and validating inputs correctly took significantly more engineering effort than training the model itself.

08. Future Improvements

Provisioning the FastAPI microservice to deploy automatically on AWS Lambda serverless endpoints using Terraform configurations.

C-Arm Medical Image Analysis Tools

Explore Source →
Python OpenCV NumPy Git
Status Completed
Role QA Intern
Duration 3 Weeks
Last Updated June 2026
Repository Explore Source →

01. Overview

Scripting tools to process and verify visual output alignment and compute difference metrics across simulator C-Arm frames.

02. Problem

Verifying visual consistency across medical simulator frames manually is inefficient. The tool needs to automate quality metric calculations between raw frames and baseline reference frames.

03. Approach

Built image loading, matrix alignment, and quality metric comparison modules using OpenCV and NumPy pixel matrix operations.

04. System Design

Image Directory Path │ ▼ OpenCV Image Loader │ ▼ NumPy Matrix Alignment │ ▼ Mask HUD Overlay (Annotation Filter) │ ▼ MSE & PSNR Quality Computations │ ▼ Validation Reports Logger

05. Engineering Decisions

Utilized OpenCV and NumPy to perform fast matrix calculations directly on pixels, keeping execution runtime minimal without heavy dependencies.

06. Challenges

Simulator HUD overlay texts (timestamps, status logs) caused false alignment error signals. Mitigated by applying OpenCV threshold masks to isolate and evaluate only structural image regions.

07. Lessons Learned

Biggest Lesson (Dirty Data): Real-world data is extremely dirty. Image verification is less about math optimization and more about proper masking and thresholding to filter out non-anatomical visual noise.

08. Future Improvements

Adding concurrent file processing loops and generating summary dashboards of simulator runs.
